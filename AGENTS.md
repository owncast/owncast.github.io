# AGENTS.md

This is the Owncast documentation and marketing website, built with **Docusaurus 3.9.1**, **React 19**, **TypeScript**, and **Tailwind CSS**. It is deployed to [owncast.online](https://owncast.online).

## Quick Reference

```bash
npm install          # Install dependencies (Node >= 20)
npm start            # Dev server with hot reload
npm run build        # Full production build (runs prebuild first)
npm run typecheck    # TypeScript type checking
```

The prebuild step fetches dynamic data from GitHub (releases, contributors, avatars, API URLs). It requires network access and may need a `GITHUB_TOKEN` env var for rate limits.

## Project Structure

```

  shared/              # Reusable content fragments included via IncludeMarkdown
  troubleshoot/        # Troubleshooting guides (also has shared/ subfolder)

src/
  components/          # React components
    landing/           # Landing page section components (hero, features, CTA, etc.)
    homepage/          # Homepage-specific sections
    chat/              # Chat UI components (uses @chatscope)
    shared/ui/         # Reusable UI primitives (button, badge, tabs, dialog, etc.)
  pages/               # Docusaurus page routes (.tsx, .mdx, .md)
  theme/               # Docusaurus theme overrides (navbar, footer, doc layout)
  css/                 # Global styles and CSS modules
  hooks/               # Custom React hooks (Matrix chat client)
  lib/                 # Utility functions and Matrix integration
  data/                # Static data files (navigation trees, directory, store items)
plugins/               # Custom Docusaurus plugins
  related-docs/        # TF-IDF-based related document recommendations
  github-info/         # Fetches GitHub repo stars and latest release
  milestones-plugin/   # Fetches GitHub milestones for roadmap display
scripts/               # Build-time scripts (fetch releases, contributors, avatars)
static/                # Static assets served as-is (images, fonts, JS, CSS)
i18n/                  # Translations for en, es, fr, de (managed via Crowdin)
```

## Key Conventions

### Documentation files

- **`.md` files** use standard CommonMark (no JSX). Use this for most docs.
- **`.mdx` files** use MDX and can import React components. Use only when needed.
- Every doc page needs frontmatter with at least `title`, `description`, and `tags`.
- Sidebar ordering is controlled by `sidebars.ts` (explicit) or `sidebar_position` in frontmatter (for autogenerated sections).
- Shared/reusable content goes in `docs/shared/` and is included via the `IncludeMarkdown` component. Shared files should have `unlisted: true` and `related: { excludeFromAll: true }` in frontmatter.
- Images for docs go in `static/images/` or `static/docs/` and are referenced with absolute paths (e.g., `/images/example.png`).

### React components

- **Prefer existing components over building from scratch.** Before creating new UI, check `src/components/shared/ui/` for pre-built primitives (button, badge, tabs, accordion, dialog, carousel, etc.) built on Radix UI. Also check the other libraries already in the project — `@chatscope/chat-ui-kit-react` for chat UI, `lucide-react` for icons, `framer-motion` for animation, `embla-carousel-react` for carousels, and `class-variance-authority` for variant-driven component styling. Use and compose what exists rather than designing and styling new elements from scratch.
- Path alias `@/` maps to `src/` (e.g., `import { Button } from "@/components/shared/ui/button"`).
- UI primitives in `src/components/shared/ui/` follow Radix UI patterns.
- Styling uses a mix of Tailwind utility classes and CSS modules (`.module.css` files colocated with components).
- The `cn()` utility from `src/lib/utils.ts` merges Tailwind classes (clsx + tailwind-merge).

### Styling

- **Prefer standalone CSS files over inline styles or Tailwind utility classes.** Write styles in colocated `.module.css` files or in `src/css/`. Tailwind is available in the project but CSS files are the preferred approach for new work.
- Dark mode only (light/dark toggle is disabled).
- Tailwind is configured with `selector` dark mode strategy on `[data-theme="dark"]`.
- Tailwind preflight is disabled to avoid conflicts with Docusaurus's Infima CSS framework.
- Brand colors: primary (purple) and secondary (blue), each with 5 tones defined in `tailwind.config.js`.
- When writing CSS, avoid conflicting with Infima's class names.

### Redirects

- Redirect rules are defined inline in `docusaurus.config.ts` in the `ALL_REDIRECTS` array at the top of the file.
- Supports both wildcard pattern redirects and static path redirects.
- Additional redirects are in `redirects.json` (loaded by `docusaurus.config.ts`).

### Internationalization

- 4 locales: English (default), Spanish, French, German.
- Translations live in `i18n/<locale>/` directories.
- React strings use `translate()` for props and `<Translate>` for JSX content.
- Crowdin integration handles translation management (`crowdin.yml`).
- When adding a new locale, update: `docusaurus.config.ts`, `src/theme/NavbarItem/LocaleDropdownNavbarItem/index.tsx`, and `src/theme/Root.tsx`.

## Build Pipeline

The `prebuild` script runs before `build` and executes in order:

1. `scripts/update-api-urls.js` — Updates `api-config.json` with latest release tag
2. `scripts/generate-contributors.js` — Fetches and processes GitHub contributors
3. `scripts/download-avatars.js` — Downloads contributor avatar images
4. `scripts/fetch-releases.js` — Fetches GitHub releases and generates markdown in `releases/`

These scripts cache results (`.contributors-cache.json`, `.releases-cache.json`) and use ETags to minimize API calls.

## Custom Plugins

### `plugins/related-docs/`
Computes related documents using TF-IDF text similarity, tag overlap, heading overlap, and same-section boosting. Scoring weights are configured in `docusaurus.config.ts`. Per-page overrides are available via the `related` frontmatter field (`include`, `exclude`, `max`, `minScore`, `disable`, `excludeFromAll`).

### `plugins/github-info/`
Fetches repository star count and latest release version from the GitHub API at build time. Makes this data available globally via `useGlobalData()`.

### `plugins/milestones-plugin/`
Fetches GitHub milestones and their associated issues. Powers the roadmap page. Uses semver parsing to sort milestones and identifies the "current" milestone by most recent activity.

## Common Tasks

### Adding a new documentation page
1. Create a `.md` file in the appropriate `docs/` subdirectory.
2. Add frontmatter (`title`, `description`, `tags`, optionally `sidebar_position`).
3. If the section uses explicit sidebar ordering, add the doc ID to `sidebars.ts`.

### Adding a new page (non-docs)
Create a `.tsx`, `.mdx`, or `.md` file in `src/pages/`. It will be served at the corresponding URL path.

### Adding a redirect
Add an entry to the `ALL_REDIRECTS` array in `docusaurus.config.ts`.

### Modifying the landing page
The homepage is composed in `src/pages/index.tsx` from components in `src/components/landing/` and `src/components/homepage/`.

### Working with the chat feature
Chat uses the Matrix protocol. Client logic is in `src/lib/matrix/` and `src/hooks/useMatrixClient.tsx`. UI components are in `src/components/chat/` using the `@chatscope/chat-ui-kit-react` library.

## Things to Watch Out For

- **Do not enable `rspackBundler`** in `docusaurus.config.ts` — it causes 404 issues with this project (noted in config comments).
- **Markdown format is set to `detect`**: `.md` files are parsed as CommonMark (no JSX allowed), `.mdx` files are parsed as MDX. If a `.md` file needs JSX, rename it to `.mdx`.
- **The blog preset is disabled.** Releases and news use separate `@docusaurus/plugin-content-blog` instances with their own `id`, `path`, and `routeBasePath`.
- **GitHub API rate limits**: Build scripts that fetch from GitHub may fail without a `GITHUB_TOKEN` environment variable, especially in CI.
- **Cache files** (`.contributors-cache.json`, `.releases-cache.json`, `.contributors-data.json`) are generated artifacts — do not edit them manually.
- **`sidebars-generated.js`** is a generated file — do not edit. The source of truth is `sidebars.ts`.
