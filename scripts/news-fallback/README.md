# news-fallback

A pre-build script that guarantees the site's news section is never empty.

## Why

The Owncast server admin pulls community news from the JSON feed generated by
the `content/news` section of this site. If every news post is a draft (the
default for hand-authored posts) or there are none, that feed is empty and
server operators see nothing.

## What it does

When run, the script:

1. Scans `content/news/*.md` for a **published** post (not a draft, not
   future-dated). If one exists, it does nothing.
2. Otherwise it fetches the most recent entry from the Owncast newsletter at
   <https://owncast.ghost.io/rss/>, converts the HTML to Markdown, and writes it
   to `content/news/<slug>.md` as a published post tagged `autogenerated: true`.

Any network or parsing error is logged and the script exits `0`, so it can
never break a deploy.

## When it runs

It's wired into the build in two places:

- `.github/workflows/main.yml` runs it before the Hugo build. That workflow
  builds and deploys to GitHub Pages on every push to `master` and on a
  schedule every six hours, so a stale or empty news feed is refreshed
  automatically.
- `package.json` `prebuild`, so a local `npm run build` runs it too.

## Notes

- Generated posts are build artifacts. They are NOT committed to the repo. Each
  scheduled build with no published news regenerates the post from the latest
  newsletter, so the fallback always reflects the most recent issue. As soon as
  a real, human-authored published post exists, generation stops.
- Dependencies (`turndown`, `gray-matter`) live in the root `package.json` so
  the workflow's `npm install` picks them up.
